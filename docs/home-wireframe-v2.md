# DK App Tools Home Wireframe v2

## Purpose

The home page is the entry point for the DK App Tools brand.

It must present DK App Tools as:
- an independent software brand
- a growing app catalog
- a source of useful articles
- a place for community feedback and future app ideas

The home page must not feel like a landing page for only Offline FileSecure.

## Page structure

### 1. Header

Navigation:
- Apps
- Categories
- Articles
- Community
- Support

Goal:
Give visitors quick access to the main parts of the website.

### 2. Compact Hero

Role:
Introduce DK App Tools as a brand, not a single product.

Suggested headline:
Practical software. Useful content. Built independently.

Suggested supporting copy:
DK App Tools builds focused apps, practical guides, and clear software resources for users who prefer simple tools with honest behavior.

Primary actions:
- Explore apps
- Read articles
- Suggest an idea

Visual role:
Clean, modern, technology-focused, but not oversized or empty.
The hero should not consume the whole page.

### 3. Hub Overview

Three main hub cards:

1. Apps
   Purpose: Browse DK App Tools products.
   Link: /apps

2. Articles
   Purpose: Read guides, privacy notes, updates, and practical software content.
   Link: /articles

3. Community
   Purpose: Polls, feedback, and future app ideas.
   Link: /community

Goal:
Immediately explain that the website is more than one product.

### 4. Featured Apps

Layout:
Catalog-style section, not a huge product landing block.

Cards:
- Offline FileSecure
  Status: First app / Android / Privacy & Security / File Tools
  Link: /apps/offline-filesecure

- Coming soon slot
  Status: Future app

- Community-voted idea slot
  Status: Idea stage

Goal:
Show the app catalog concept without pretending there are many finished products yet.

### 5. Browse Categories

Initial category cards:
- Privacy & Security
- File Tools
- Productivity
- Android Tools
- Future Experiments

Goal:
Prepare the site for future products and let users browse by purpose.

### 6. Latest Articles

Article cards:
- How to protect private files on Android
- Why local-first apps matter
- Cloud vs local storage for sensitive files

Goal:
Create shareable and SEO-friendly content areas.
The articles section should feel like a real content engine, not a dead placeholder.

### 7. Community / Poll Preview

Suggested headline:
What should DK App Tools build next?

Poll-style option cards:
- Privacy tool
- File utility
- Productivity app
- Android helper

Goal:
Show that visitors can influence future direction.

### 8. Why DK App Tools

Principle cards:
- Practical first
- Clear privacy
- Independent development
- Honest product pages

Goal:
Explain the brand philosophy without over-marketing.

### 9. Footer

Footer links:
- Apps
- Categories
- Articles
- Community
- Privacy
- Support

Goal:
Support discovery and trust.

## Layout rules

- Do not use oversized empty headings for every section.
- Use mixed section types: hub cards, catalog cards, article cards, poll cards, principle cards.
- Keep the home page scannable.
- Each section must have a clear purpose.
- Offline FileSecure should appear as the first app, not as the whole website.
- The page should feel like a modern software hub.

## Visual direction notes

Preferred:
- modern tech/product feel
- structured layout
- strong content hierarchy
- dark or graphite base with controlled accent color
- premium but not fantasy
- enough density to feel useful

Avoid:
- Lineage2/server style
- white news-site style
- random cards
- giant poster layout
- single-product landing page behavior
